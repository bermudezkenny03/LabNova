# Documentación Técnica de Lab Nova

Esta carpeta contiene la documentación principal del proyecto Lab Nova para desarrolladores, QA y operaciones.

## Estructura

- `architecture.md` – Visión general de la arquitectura, tecnologías y estructura de carpetas.
- `setup.md` – Instrucciones para instalar y ejecutar el proyecto localmente.
- `api.md` – Documentación de los endpoints REST disponibles en el backend.
- `deployment.md` – Flujo de despliegue actual con Vercel y configuración relevante.
- `testing.md` – Guía de pruebas existentes y recomendaciones para validar calidad.
- `quality-guidelines.md` – Buenas prácticas de calidad, seguridad y accesibilidad.

## Cómo usar este documento

1. Comienza por `setup.md` si necesitas levantar el proyecto localmente.
2. Usa `architecture.md` para entender la separación frontend/backend.
3. Consulta `api.md` para conocer los endpoints disponibles y su propósito.
4. Revisa `deployment.md` antes de publicar en Vercel.
5. Sigue `testing.md` y `quality-guidelines.md` para mantener los estándares de calidad.
